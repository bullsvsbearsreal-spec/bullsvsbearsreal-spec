## Authentication & User Tables

### users (NextAuth core)
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | UUID | PK | Auto-generated user ID |
| name | TEXT | | Display name |
| email | TEXT | UNIQUE | User email address |
| email_verified | TIMESTAMPTZ | NULL | When email was verified (NULL = unverified) |
| image | TEXT | NULL | Avatar URL (NULL for data URIs >32KB — Vercel header limit) |
| password_hash | TEXT | NULL | bcrypt hash (salt rounds: 12). NULL for OAuth-only accounts |
| role | TEXT | DEFAULT 'user' | 'user', 'admin', or 'advisor' |

Seed admin: ocelotcex1638a@gmail.com → role='admin'

### accounts (NextAuth OAuth links)
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| user_id | UUID | FK → users.id | Owning user |
| provider | TEXT | | OAuth provider (google, discord, twitter) |
| provider_account_id | TEXT | | Provider's user ID |
| type | TEXT | | Account type (oauth) |
| access_token | TEXT | | OAuth access token |
| refresh_token | TEXT | | OAuth refresh token |
| expires_at | INTEGER | | Token expiry (Unix seconds) |
| scope | TEXT | | OAuth scopes granted |
| id_token | TEXT | | OIDC ID token |
| session_state | TEXT | | Provider session state |

### sessions (NextAuth session tokens)
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| session_token | TEXT | UNIQUE | Random session identifier |
| user_id | UUID | FK → users.id | Session owner |
| expires | TIMESTAMPTZ | | Session expiry time |

### verification_tokens (NextAuth)
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| identifier | TEXT | | Email or phone number |
| token | TEXT | | Verification token |
| expires | TIMESTAMPTZ | | Token expiry |

## Two-Factor Authentication Tables

### email_verification_codes
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment ID |
| user_id | UUID | FK → users.id | User being verified |
| email | TEXT | | Email address |
| code | TEXT | | 6-digit verification code |
| expires_at | TIMESTAMPTZ | | 15 minutes from creation |
| used | BOOLEAN | DEFAULT false | Claimed flag |

Index: `idx_evc_user (user_id, used)`

### twofa_login_codes
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment ID |
| email | TEXT | | User email |
| code | TEXT | | 6-digit 2FA code |
| expires_at | TIMESTAMPTZ | | 15 minutes from creation |
| used | BOOLEAN | DEFAULT false | Atomic claim flag (prevents TOCTOU race) |

### user_2fa
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| user_id | UUID | PK, FK → users.id | One row per user |
| totp_secret | TEXT | | Base32-encoded TOTP secret (20 bytes) |
| totp_enabled | BOOLEAN | DEFAULT false | Whether TOTP is active |
| email_2fa_enabled | BOOLEAN | DEFAULT false | Whether email 2FA is active |
| backup_codes | TEXT[] | | Array of 8 backup codes (4 random bytes each, uppercase hex) |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | Setup time |
| updated_at | TIMESTAMPTZ | DEFAULT NOW() | Last modification |

### password_reset_tokens
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment ID |
| token | TEXT | | SHA256 hex token |
| email | TEXT | | Target email |
| expires_at | TIMESTAMPTZ | | 1 hour from creation |
| used | BOOLEAN | DEFAULT false | Claimed flag |

Index: `idx_prt_email (email)`

## Market Data Snapshot Tables

### funding_snapshots
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| symbol | TEXT | | Normalized symbol (BTC, ETH, etc.) |
| exchange | TEXT | | Exchange name |
| rate | REAL | | Funding rate (native interval %) |
| predicted | REAL | NULL | Predicted next funding rate |
| mark_price | REAL | NULL | Mark price at snapshot time |
| ts | TIMESTAMPTZ | | Snapshot timestamp |

Indexes:
- `idx_funding_sym_ts (symbol, ts)`
- `idx_funding_sym_ex_ts (symbol, exchange, ts)`
- `idx_funding_ts_symbol (ts, symbol)`
- `idx_funding_mark_price (mark_price)` — used for price lookups

Population: Cron every 10 minutes, top 200 symbols by exchange count.
Retention: 30 days (auto-pruned).

### oi_snapshots
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| symbol | TEXT | | Normalized symbol |
| exchange | TEXT | | Exchange name |
| oi_usd | REAL | | Open interest in USD |
| ts | TIMESTAMPTZ | | Snapshot timestamp |

Indexes:
- `idx_oi_sym_ts (symbol, ts)`
- `idx_oi_sym_ex_ts (symbol, exchange, ts)`
- `idx_oi_ts_symbol (ts, symbol)`

Population: Cron every 10 minutes, top 200 symbols.
Retention: 30 days.

### spread_snapshots
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| symbol | TEXT | | Normalized symbol |
| spread_usd | REAL | | High price - low price |
| spread_pct | REAL | | Spread as percentage |
| high_exchange | TEXT | | Exchange with highest price |
| low_exchange | TEXT | | Exchange with lowest price |
| high_price | REAL | | Highest price |
| low_price | REAL | | Lowest price |
| exchange_count | INTEGER | | Number of exchanges with this pair |
| ts | TIMESTAMPTZ | | Snapshot timestamp |

Index: `idx_spread_sym_ts (symbol, ts)`

### arb_opportunities
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| symbol | TEXT | | Trading pair |
| spread_usd | REAL | | Current spread USD |
| spread_pct | REAL | | Current spread % |
| high_exchange | TEXT | | Exchange with higher price |
| low_exchange | TEXT | | Exchange with lower price |
| max_spread_usd | REAL | | Peak spread USD during opportunity |
| max_spread_pct | REAL | | Peak spread % |
| opened_at | TIMESTAMPTZ | | When opportunity was first detected |
| closed_at | TIMESTAMPTZ | NULL | When opportunity closed (NULL = still open) |
| status | TEXT | DEFAULT 'open' | 'open' or 'closed' |

Indexes:
- `idx_arb_opp_sym_status (symbol, status)`
- `idx_arb_opp_opened (opened_at)`

### liquidation_snapshots
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| symbol | TEXT | | Trading pair |
| exchange | TEXT | | Source exchange |
| side | TEXT | | 'long' or 'short' |
| price | REAL | | Bankruptcy/liquidation price |
| quantity | REAL | | Position size |
| value_usd | REAL | | USD value of liquidation |
| ts | TIMESTAMPTZ | | Event timestamp |

Unique index: `idx_liq_dedup (symbol, exchange, side, price, ts)` — prevents duplicate ingestion.

Population: Continuous via WebSocket ingestion + REST polling cron.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| user_id | UUID | FK → users.id | Watchlist owner |

### alert_notifications
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | SERIAL | PK | Auto-increment |
| user_id | UUID | FK → users.id | Alert recipient |
| alert_id | TEXT | | Alert rule ID |

- `idx_alert_notif_sent_channel (sent_at, channel)`

## API & Integration Tables

### api_keys
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | UUID | PK | Key identifier |
| user_id | UUID | FK → users.id | Key owner |
| key_hash | TEXT | UNIQUE | SHA256(raw key) — raw key never stored |
| key_prefix | TEXT | | First 8 chars of key (display only) |
| name | TEXT | | User-assigned key name |
| tier | TEXT | DEFAULT 'free' | 'free' or 'pro' |
| is_active | BOOLEAN | DEFAULT true | Whether key is active |
| last_used_at | TIMESTAMPTZ | NULL | Last API call timestamp |
| requests_today | INTEGER | DEFAULT 0 | Daily request counter |
| created_at | TIMESTAMPTZ | DEFAULT NOW() | Creation time |

Indexes:
- `idx_api_keys_hash (key_hash)` — fast lookup
- `idx_api_keys_user (user_id)` — list user's keys

Key format: `ih_` + 32 random bytes (base62), max 5 keys per user.

## Cache Tables

### api_cache (L2 persistent cache)
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| key | TEXT | PK | Cache key (e.g., 'top500_symbols', 'fear_greed') |
| data | JSONB | | Cached response payload |
| expires_at | TIMESTAMPTZ | | Cache expiry (entry-specific TTL) |

Index: `idx_api_cache_expires_at (expires_at DESC)`

Purpose: Survives Vercel Edge cold starts (L1 memory cache is lost). TTL varies by data type:
- top500 symbols: 1800s (30 min)
- fear & greed: 3600s (1 hr)
- dominance: 600s (10 min)
- top movers: 1800s (30 min)
